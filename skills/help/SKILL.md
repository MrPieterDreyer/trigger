# Help

When the user says **trigger help** or **what can trigger do**:

**Getting started** — *trigger new project*: initialize a Trigger project. *trigger status* / *trigger progress*: current position.

**Planning** — *trigger plan phase N*: create the implementation plan for phase N.

**Building** — *trigger execute phase N* / *trigger build phase N*: full build-review pipeline for a phase. *trigger build task N*: single task.

**Session** — *trigger pause*: save progress. *trigger resume*: pick up where you left off.

**Configuration** — *trigger config*: view or edit project configuration (`trigger.json`).

Full documentation: the plugin's `README.md` and `rules/trigger-core.mdc`.
